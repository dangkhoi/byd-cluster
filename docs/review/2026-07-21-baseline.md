## 3. WHERE THE CODE ALREADY VIOLATES ITS OWN RULES

### 3.1 Two remaining blind display-scans (I2) — **P0/P1**
- **`CastShell.restoreFullscreenOnMain:73`** — `StackParse.of(ents, pkg).filter { it.displayId >= 1 }` then `am display move-stack $it 0`. This is **the exact `displayId >= 1` blind-scan shape that froze the Dudu launcher** (CLAUDE.md §4). It is narrowed by package, but it is **not** filtered by `isStandard` and **not** filtered by `!isPinned` — so a `pinned` stack of that same package on a virtual display gets dragged to display 0. **[P] AOSP `ActivityStack.reparent()` (android-10.0.0_r47, ~1272-1298), order verbatim:** `removeFromDisplay(); … activityDisplay.addChild(this, onTop ? POSITION_TOP : POSITION_BOTTOM); if (!displayRemoved) { postReparent(); }` — and **[P] `ActivityDisplay.addStackReferenceIfNeeded()` (lines 1019-1072) throws `IllegalArgumentException`** when a home/recents/**pinned**/split-primary stack of that type already exists on the target display. Throwing *after* `removeFromDisplay()` and *before* `postReparent()` is precisely the orphaned-stack / never-resumed-launcher failure. Called from 7 sites including `stop():457`, `evictVd:198`, warm-switch `:328/:351`. **This is the single highest-severity item in the baseline.**
- **`ClusterCast.reconcileOnStart:827-828`** — `StackParse.of(...).firstOrNull { it.displayId == 0 && (it.isFreeform || it.isPinned) }` then routes into `restoreFullscreenOnMain`, inheriting the above. Runs at **BOOT** and after **every app update** (`RebindReceiver:81`) with nobody pressing anything.

### 3.2 Broken undo paths (I4/I5)
1. **`rollback():486`** — `adb.shell(CastShell.pipCmd(pipBlockedPkg, "allow"))`. Hard-coded `"allow"`, **ignores `pipPrevMode`**, and **does not clear `pipBlockedPkg`/`pipPrevMode`**. The spec's Reviewer Log Pass-2 claims this exact defect was fixed ("rollback() còn trả PIP bằng 'allow' cứng" — `cluster-cast-v036.html` §Reviewer Log). **It was not.** A CLAUDE.md §8 failure: the claimed patch has no call site. **[P]**
2. **`svcName` is RAM-only and only ever assigned inside `cast()`** (`:238` default `"AutoContainer"`, assigned `:374`). `stop():469`, `rollback():484`, and **`reconcileOnStart():836`** all call `sc()`. `reconcileOnStart` **never sets it at all**. On a DiLink5 (`svcName = "auto_container"`, `ClusterProfile.kt:60`) every one of those teardowns is fired at a service that does not exist — silently, exit 0, cluster stays projected. This is verbatim the failure mode CLAUDE.md §7 warns about. **[P]**
3. **`ensureFreeformSeed` (`CastShell.kt:47-54`)** writes three `Settings.Global` keys including `force_resizable_activities 1`, which after the next power cycle makes **every app on the vehicle** resizable, and the KDoc itself states there is no in-app undo (`:43-44`). No durable marker, no UI toggle. The field car already has all three at `1` (`00-state.txt:13-15`). **[P]** The spec lists "nút hoàn tác force_resizable_activities" as an unclosed Open Question.
4. **Animation scales** — `Prefs.animOpt` defaults **true**, so `MainActivity.kt:135` writes `0.5` globally on every app open. `cast():379` then reads `window_animation_scale` as `savedAnim` — which by then is **our own 0.5**, not the user's original. `savedAnimSafe:59` only protects against restoring `0`. Net: the user's original value is unrecoverable after the first launch. Field dump shows `window=0, transition=0` left behind (`00-state.txt:16-17`). **[P]**
5. **No watchdog for the cast app dying.** `ISSUES.md:30-33` asks for one explicitly. `grep` finds no polling loop while `casting==true`. If the projected app is killed or unplugged (exactly what happened during log collection), the cluster keeps projecting with `mHasContent=false` (`05-display.txt:114`), `03-vd-stacks.txt` is **empty**, and density=200/overscan=(536,224,88,336) sit on the VD (`02-summary.txt:2,7`) until the user presses TẮT. **[P]**
6. **`reconcileOnStart` cannot detect "cluster is projecting but empty."** It sends the teardown sequence only `if (onVd.isNotEmpty())` `:834`. The dirty-VD probe `:815-816` compares `logicalSize` vs `realSize` — it catches leaked **`wm size`** but **not** leaked density or overscan, which are the two things actually observed leaked in the field. So the exact state captured in `carlog-2026-07-21` would **not** be cleaned by a restart. **[P]**

### 3.7 Smaller, still real
- **`DisplayParse.appWindowFrame:73-83` is display-blind and takes the first matching window.** It scans `dumpsys window windows` for any header containing `"$pkg/"`. Sub-windows (`SurfaceView - pkg/…`) and windows on display 0 both match. `overscanVerified` — the measurement that decides whether to escalate to the destructive `wm size` tier — is built on it. **[L]** that this misreads for any app with a window on both displays; **closing observation:** capture `dumpsys window windows` (never yet collected — `ISSUES.md:38`) while AA is on the cluster and count matching headers.
- **`overscanVerified:156-159` reads the frame back with no settle delay** after `wm overscan`. Relayout is asynchronous; a stale read returns "app ignored the inset" → escalate to `wm size` → the user's off-centre frame is silently replaced by a centred letterbox. **[L]**
- **`AppScale` stores absolute pixels** (`rectL/T/R/B`) against `clusterRef()` (`ClusterCastActivity.kt:401-404`), which falls back to the **profile's 1920×720** before the first ever cast. `boundsOn(w,h):24` does **not** rescale to the actual cluster. A profile shared with an SL6 owner (S9) carries the cluster geometry but **not** the per-app rects, and any rect adjusted before the first cast is computed against a guessed size. Per-app vs per-model is split in the wrong place.
- **`DeadReckonModule.selfTest` calls `MockLoc.start(ctx)` then `MockLoc.stop(ctx)`** (`DeadReckonModule.kt:36`) — i.e. `addTestProvider`/`removeTestProvider` on the live `GPS_PROVIDER`. - **`SpeedProvider.bypass():49-56` returns `getPackageName() = "com.byd.dashcast"`** and calls `VMRuntime.setHiddenApiExemptions` `:59-64` — the app impersonates another package to pass BYD HAL permission checks. - **`ClusterCast.kt` is 924 LOC**, over the repo's own 500-LOC guardrail (global CLAUDE.md §4.1). `CastShell.kt` was the extraction; it stopped at 200 LOC.
